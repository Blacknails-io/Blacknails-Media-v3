# Project Documentation Maintainer

This skill ensures that the system's documentation always reflects the actual implemented features and architecture.

## When to use this skill
- ALWAYS use this skill immediately after implementing a new feature, fixing a major structural bug, or changing the system's architecture.
- Use this skill when modifying database schemas, API routes, or the media import pipeline.

## How to use it
- Please refer to the documentation update guidelines located in [guidelines.md](resources/guidelines.md).
