# Naming Conventions Guideline

Rules for naming variables, functions, classes, and other identifiers in this project.

## Naming Conventions
- **camelCase**: Use for variables, function names, properties, and method names (e.g., `getUserData`, `isVerified`).
- **PascalCase**: Use for class names, interfaces, types, enums, and components (e.g., `UserService`, `UserRole`, `HeaderComponent`).
- **UPPERCASE_WITH_UNDERSCORES**: Use for global constants, configuration constants, and environment variables (e.g., `MAX_RETRY_COUNT`, `API_URL`).
- **Descriptive Names**: Avoid short, generic names like `d`, `temp`, `data` (unless in trivial loop indices like `i`). Be explicit (e.g., `lastUpdatedTimestamp` instead of `date`).
