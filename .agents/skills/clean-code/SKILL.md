# Clean Code Skill

This skill ensures clean code standards across the project, including naming conventions, file organization, and SOLID principles when writing or refactoring code.

## When to use this skill
- Writing new code
- Refactoring existing code
- Reviewing code for compliance with clean code standards

## When NOT to use this skill
- Writing or refactoring code that is not related to clean code standards
- The user explicitly asks you not to use this skill

## Standards and Best Practices

To keep this codebase high quality and consistent, follow these modular guidelines:

- **[Naming Conventions](./references/naming-conventions.md)**: Rules for variables, functions, classes, and constants naming.
- **[SOLID Principles](./references/solid-principles.md)**: Object-oriented and design structural guidelines to ensure modularity.
- **[JS / TypeScript Best Practices & Error Handling](./references/typescript-best-practices.md)**: Code safety, strict typing rules, array manipulation, and proper error management.
