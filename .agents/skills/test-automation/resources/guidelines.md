# Test Automation & QA Guidelines

This skill defines the testing strategies, frameworks, and verification protocols for **Blacknails-Media-v3**.

## 1. End-to-End (E2E) Testing with Playwright

Playwright is configured under `/client` (see [playwright.config.ts](file:///srv/storage/ai-lab/Blacknails-Media-v3/client/playwright.config.ts)). E2E tests are stored in `/client/tests`.

- **Selector Best Practices**:
  - Prefer accessible locators: `page.getByRole()`, `page.getByText()`, `page.getByLabel()`.
  - Avoid fragile CSS selectors that depend on layout structure. If needed, add unique `data-testid` attributes.
- **Handling UI Transitions**:
  - Since we use Framer Motion and Atropos, ensure you wait for animations to complete before capturing screenshots or checking layout state. Use `page.waitForSelector()` or check for opacity/position transitions.
  - Disable animations in Playwright configuration or mock animations during visual regression checks.

## 2. Test Fixtures & Media Setup

- To test upload and gallery render features, use the test files generated by the workspace scripts:
  - [generate_test_files.sh](file:///srv/storage/ai-lab/Blacknails-Media-v3/generate_test_files.sh) (creates mocked images/videos).
  - [generate_manual_fixtures.sh](file:///srv/storage/ai-lab/Blacknails-Media-v3/generate_manual_fixtures.sh).
- Do not check in large binary test files to Git; generate them locally using the scripts before running tests.

## 3. Running and Debugging Tests

- **Running client tests**: `npm run test:e2e --workspace=blacknails-media-v3-client` or inside the `client` directory: `npx playwright test`.
- **Running server tests**: run mocha/jest/vitest unit tests for application services and domain models.
- **Test environment**: Always clean up test files (database changes, imported media files in testing libraries) in the `afterAll` or `teardown` hooks to ensure hermetic and reproducible test runs.
