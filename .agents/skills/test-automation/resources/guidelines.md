# Test Automation & QA Guidelines

This document defines the testing strategies, frameworks, and verification protocols for **Blacknails-Media-v3** E2E testing.

---

## 1. End-to-End (E2E) Testing with Playwright

Playwright is configured under the client package (see **[playwright.config.ts](../../../client/playwright.config.ts)**). E2E test files are stored in `client/tests/`.

### Selector Best Practices
- **Prefer accessible locators**: `page.getByRole()`, `page.getByText()`, `page.getByLabel()`.
- **Avoid fragile CSS selectors**: Do not rely on layout structures that can change. If needed, add unique `data-testid` attributes to HTML components.

### Handling UI Transitions
- **Animations**: Since we use Framer Motion and Atropos, ensure you wait for animations to complete before capturing screenshots or checking layout state. Use `page.waitForSelector()` or check for opacity/position transitions.
- **Visual Regression Checks**: Disable animations in Playwright configuration or mock animations during visual regression checks.

---

## 2. Test Fixtures & Media Setup

- To test upload and gallery render features, use the test files generated by the workspace scripts in the root directory:
  - **[generate_test_files.sh](../../../generate_test_files.sh)** (creates mocked images/videos).
  - **[generate_manual_fixtures.sh](../../../generate_manual_fixtures.sh)**.
- **NEVER** check in large binary test files to Git; generate them locally using the scripts before running tests.

---

## 3. Running and Debugging Tests

- **Running client E2E tests**:
  - From the workspace root: `npm run test:e2e --workspace=blacknails-media-v3-client`
  - Or inside the `client` directory: `npx playwright test`
- **Test environment cleanup**: Always clean up test files (database changes, imported media files in testing libraries) in the `afterAll` or `teardown` hooks to ensure hermetic and reproducible test runs.
